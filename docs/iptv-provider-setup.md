# IPTV Provider Setup

Complete reference for running GameStop Kenya IPTV — from first login to full provider integration.

---

## 1. Adding a Super Admin

Super admin is configured entirely through environment variables. No database or code change is needed.

Create or edit `.env.local` in the project root:

```env
SUPER_ADMIN_NAME=Your Name
SUPER_ADMIN_PHONE=0712345678
SUPER_ADMIN_EMAIL=you@gamestop.co.ke
SUPER_ADMIN_PASSWORD=a-very-strong-secret
```

- Either `SUPER_ADMIN_PHONE` or `SUPER_ADMIN_EMAIL` is required (both is fine).
- `SUPER_ADMIN_PASSWORD` is required.
- Restart the dev server or redeploy for changes to take effect.
- Log in at `/admin` with the phone or email + password above.

On Vercel (or any hosting platform), set the same four keys in the environment variable settings panel instead of a file.

**There is only one super admin.** To create sub-admins, log in as super admin, go to the Admin Team panel, and add them from the dashboard UI.

## 2. Admin roles

| Role | What they can do |
| --- | --- |
| `super_admin` | Everything — manage all subscriptions, manage admins, generate access codes, grant access, impersonate users |
| `admin` | View and manage subscriptions assigned to them; cannot manage other admins or generate codes |

## 3. Giving someone IPTV access

### Option A — Grant directly (no payment required)

Super admin only. In the admin dashboard under **Grant Access**:

1. Enter the customer's phone number (Kenyan format, e.g. `0712345678`).
2. Enter their name.
3. Pick the plan (1 Week / 1 Month / 3 Months / 12 Months / 24 Months).
4. Click **Grant & Activate**.

The subscription is created and activated instantly. The dashboard shows the member's access code — share it with them so they can log in at `/movies/login`.

### Option B — Generate shareable access codes

Super admin only. In the admin dashboard under **Access Code Generator**:

1. Select the plan.
2. Enter quantity (1–50 per batch).
3. Optionally add a note (e.g. "Promo March 2026").
4. Click **Generate Codes**.

Codes are in the format `GSTV-XXXX-XXXX`. Distribute them to customers. When a customer redeems a code they receive an active subscription for the chosen plan duration.

Codes table shows all generated codes, their plan, when they were created, and whether they have been redeemed.

### Option C — Customer pays via M-Pesa

Standard flow — customer goes to `/iptv`, picks a plan, pays via STK Push, subscription activates automatically after payment confirmation.

---

## 4. Member login flow

Once a customer has access (via any method above):

1. They go to `/movies/login`.
2. Enter their **phone number** + **8-character access code**.
3. They land on the member hub at `/movies`.

From the hub they can:

- Browse and watch VOD (movies, series).
- Watch Live TV (`/live`) — 8,000+ free iptv-org channels.
- Use their Xtream credentials in any external player (VLC, TiviMate, Kodi, Smart TV app).

---

## 5. Impersonating a user (Login as User)

Super admin only. In the admin dashboard under **Login as User**:

1. Enter the customer's phone number.
2. Click **Login as that User**.
3. A link appears — open it (in a new tab or send to yourself on another device) to be immediately logged in as that member.

Useful for debugging a customer's experience without needing their access code.

## 7. IPTV credentials (Xtream Codes)

Every activated subscription gets a set of credentials:

- **M3U URL** — paste into any M3U player
- **Xtream host / username / password / port** — use in TiviMate, Kodi, Smart TV apps

These are generated by the provisioning system (see section 9).

## 8. Live TV channels (free / iptv-org)

The platform includes 8,000+ free channels sourced from [iptv-org](https://iptv-org.github.io/) without any provider contract. These work out of the box.

Available in the member hub at `/live`, and as a horizontal rail on the hub homepage.

Categories included: Sports, Kenya, News, Entertainment, Movies, and more.

To add more categories, edit `src/lib/iptv-org.ts` → `CHANNEL_CATEGORIES`.

## 9. Provisioning modes

When a subscription activates (paid or granted), the app calls the provisioning layer to generate IPTV credentials.

### Local mode (default)

```env
IPTV_PROVISIONING_MODE=local
```

Generates fictional-but-valid Xtream credentials stored in `iptv_credentials` in Supabase. Playback works within the member hub (HLS player embedded in browser). Good for getting started without a provider panel.

### Webhook mode (real provider)

```env
IPTV_PROVISIONING_MODE=webhook
IPTV_PROVISIONING_URL=https://your-provider.com/provision
IPTV_PROVISIONING_TOKEN=shared-secret
```

On activation the app POSTs to your provisioning endpoint:

```json
{
  "subscriptionId": "IPTV-ABC123",
  "planId": "12mo",
  "planName": "12 Months",
  "months": 12,
  "amountKes": 14999,
  "phone": "254712345678",
  "customerName": "John Doe",
  "mpesaReceipt": "QHG1234XYZ",
  "expiresAt": "2027-03-10T12:00:00.000Z",
  "requestedFormat": "xtream"
}
```

Your endpoint returns real Xtream credentials:

```json
{
  "credentials": {
    "xtreamHost": "https://panel.provider.com",
    "xtreamPort": 443,
    "xtreamUsername": "user123",
    "xtreamPassword": "pass456",
    "m3uUrl": "https://panel.provider.com/get.php?username=user123&password=pass456&type=m3u_plus"
  }
}
```

These are stored in `iptv_credentials` and shown to the customer.

## 10. What the provider must supply

For full 20,000+ channel catalog:

| Item | Detail |
| --- | --- |
| Live channels | HLS/DASH URL, channel name, logo |
| Movies / series | HLS/DASH URL or Cloudflare Stream UID, plus metadata |
| Sports events | Event title, teams, start time, licensed feed URL |
| Account provisioning | Xtream-compatible provisioning API (webhook mode) |

## 11. Database tables

| Table | Purpose |
| --- | --- |
| `iptv_subscriptions` | All subscriptions (pending, active, expired) |
| `iptv_credentials` | Xtream credentials per subscription |
| `iptv_access_codes` | Generated voucher codes (GSTV-XXXX-XXXX) |
| `iptv_catalog_items` | VOD catalog (channels, movies, series, events) |
| `iptv_playback_sources` | Playback URLs per catalog item |
| `iptv_live_events` | Scheduled live events (e.g. UCL) |

SQL for `iptv_access_codes`:

```sql
create table iptv_access_codes (
  code text primary key,
  plan_id text not null,
  plan_name text not null,
  created_by_admin_id text not null,
  created_at timestamptz not null default now(),
  expires_at timestamptz,
  redeemed_at timestamptz,
  redeemed_by_phone text,
  subscription_id text,
  note text
);
```

See [schema.sql](../supabase/schema.sql) for the full catalog schema and table definitions.

## 12. Champions League / sports events

The app has a placeholder slug `uefa-champions-league-night`. To activate it:

1. Insert a `sports_event` row in `iptv_catalog_items` with that slug.
2. Add the match schedule in `iptv_live_events`.
3. Add the licensed HLS feed in `iptv_playback_sources`.

Or set the env var shortcut for a single event:

```env
IPTV_UCL_EVENT_HLS_URL=https://provider.com/events/ucl.m3u8
IPTV_UCL_EVENT_START_AT=2026-04-07T19:00:00Z
IPTV_UCL_HOME_TEAM=Arsenal
IPTV_UCL_AWAY_TEAM=Barcelona
IPTV_UCL_VENUE=Emirates Stadium
```

## 14. Recommended rollout order

1. Set `SUPER_ADMIN_*` env vars → redeploy → log in at `/admin`.
2. Test grant flow — grant yourself a subscription, log in at `/movies`.
3. Generate a batch of access codes, redeem one.
4. Connect Supabase (`NEXT_PUBLIC_SUPABASE_URL` etc.) so data persists across restarts.
5. Run the migration for `iptv_access_codes` and catalog tables.
6. Switch to `webhook` provisioning and point it at your real IPTV panel.
7. Bulk-load your provider's catalog into `iptv_catalog_items` + `iptv_playback_sources`.
